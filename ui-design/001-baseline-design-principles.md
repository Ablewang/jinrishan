# 001 - 基础设计原则与 UI 重构规范

## 背景
为了避免在未来的 UI 重构中出现“步子迈得太大”、“破坏原有风格”或“PC与H5未做合理区分”的问题，特制定此重构流程与基础设计规范。以后每次进行 UI 重构前，必须先在 `ui-design` 目录下创建一个新的设计文档（如 `002-xxx.md`），将所有的想法和设计原则列出，审查合理后再动手写代码。

## 核心设计原则

### 1. 尊重与继承原有设计语言
- **色彩体系**：在没有明确要求修改主题色的情况下，必须沿用原有的品牌色（当前为橙色系 `#e67e22` 等）。
- **组件形态**：保留原有的基础圆角（`border-radius`）、阴影（`box-shadow`）以及边框风格。
- **渐进式优化**：重构应侧重于修补现有界面的排版缺陷、增加交互反馈（Hover / Active）、优化间距，而不是彻底推翻重做。

### 2. PC 端与 H5 端的差异化适配
**必须通过 CSS 媒体查询（`@media (min-width: 768px)` 等）进行严格的设备区分。**

- **H5 端（Mobile First）**：
  - 默认的 CSS 代码优先服务于移动端。
  - 使用全屏宽度、单列流式布局。
  - 底部固定导航栏等移动端专属组件需考虑安全区（`safe-area-inset-bottom`）。
  - 弹窗（Modal）优先使用底部弹出的半屏抽屉（Bottom Sheet）样式。

- **PC 端（Desktop）**：
  - 媒体查询下，取消元素的 `100%` 强制宽度，设定合理的最大宽度（如 `max-width: 800px`）并居中。
  - 对于多数据列表（如菜谱、卡片），由单列（`flex-direction: column`）改为网格（`CSS Grid` 或 `flex-wrap`），充分利用宽屏横向空间。
  - 悬浮弹窗从底部抽屉恢复为经典的屏幕居中模态框（Centered Modal）。

### 3. 审查与执行流程
1. **收集需求**：明确本次要优化或重构的页面及组件。
2. **编写设计文档**：在 `ui-design` 下新建 Markdown 文档，列出重构范围、拟采用的 CSS 属性调整和媒体查询策略。
3. **自我审查**：回顾文档，判断设计是否偏离基础主题？是否在 PC 和 H5 上都做了合理的适配？
4. **执行代码**：确认无误后，再使用工具修改相应的 `.tsx` 和 `.module.css` 文件。
