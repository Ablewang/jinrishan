# V13 - System Richness & High-End Gallery (解决“太空/半成品”感)

## 核心诊断：为什么极简会变成“半成品”？
在之前的迭代中，我们为了追求“极致简约（Apple/Swiss Design）”，做了大量的“减法”：去掉了阴影、去掉了圆角、去掉了大面积的色块。
当一个系统完全由大面积的白/灰背景和简单的文字组成时，如果没有**高密度的细节**来支撑，它就会显得“空洞”和“未完成”。

高端画廊（High-End Gallery）之所以高级，不是因为它“空”，而是因为它在“留白”的同时，有极其考究的**微观细节（Micro-details）**作为骨架。

## 解决策略：“在框架内做加法”

为了消除内部系统的“廉价感/半成品感”，我们需要引入以下设计原则：

### 1. 微型排版（Micro-typography）与信息密度
- **引入 Overline（眉题）**：在主要标题上方或下方，增加字号极小（10-11px）、字间距极大（2-4px）、全大写（Uppercase）的辅助说明文字。这能瞬间增加排版的层次感。
- **强化 Metadata（元数据）展示**：将时间、难度、标签等信息，用极简的 Icon 配合特定的字体（如等宽数字、特定的颜色）进行重新编排，使其成为装饰的一部分，而不仅仅是信息。

### 2. 结构线与“骨架”（Hairline Grids）
- **显性化网格**：在极简设计中，“线”是最好的装饰。在内容块之间、卡片内部、甚至页面的边缘，使用极细的 `#E4E4E7` 甚至 `#F4F4F5` 线条。
- **分割而非包裹**：少用封闭的 `border` 将东西框起来，多用单边的 `border-bottom` 或 `border-left` 来引导视线。

### 3. 视觉锚点（Visual Anchors）与重量感
- **实心与空心的对比**：页面中必须有“压得住阵脚”的深色（或主色）块。主要行动点（如“就吃这个”）必须是沉稳的深色实心，而次要行动点是空心或文字。
- **微妙的背景反差**：利用 `#FFFFFF`（卡片/内容）和 `#F9F9FA`（页面底层）的微弱对比，形成空间的深度，而不是一整片死白。

### 4. 精致的空白状态（Designed Empty States）
- **不要真的“空”**：当没有数据（暂无推荐、没有计划）时，不能只是一句“暂无数据”。需要设计精美的 Placeholder，甚至是用虚线框、浅色 Icon 构成的“插画”感占位符。

## 执行清单 (Action Items)

1. **全局样式微调 (`index.css`)**：确保我们有全局的 `.overline` 和 `.hairline` 实用类。
2. **首页 (`Home`) 重构**：
   - 顶部的“今日推荐”增加眉题（Overline）。
   - 菜谱卡片的标签（Tags）和元信息（Meta）排列更像出版物的目录。
   - 优化横向滑动的“呼吸感”。
3. **空状态优化**：将首页的 Loading 和 Empty 状态做得更具设计感（使用极简动画或特定的 Typography）。
4. **按钮交互**：全系统的按钮遵循 Login 页面的逻辑（禁用/次要态轻盈，激活/主要态沉稳）。

## 批判性自检
- **会不会又变回复杂？** 不会。我们增加的是“线”和“小字”，以及深色块的反差。大面积的留白仍然保留。
- **是否符合“高级画廊”？** 完全符合。画廊的标签牌就是这种：极简底色、极细线条、多层次排版。
